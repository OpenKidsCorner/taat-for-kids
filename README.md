# Technology as a Tool (TaaT) for Kids

بسم الله الرحمن الرحيم

If Allah wills, this repository contains notes from our weekly sessions where kids learn about Technology as a Tool (TaaT) to serve Allah. May Allah accept this effort, forgive our shortcomings, and grant us sincerity.

## Sessions

| # | Date | Topic | Learning Outcomes | Notes |
|---|------|-------|-------------------|-------|
| 1 | April 04, 2026 | Introduction to Technology — Language, Devices & Robot Navigation (~4.25 hrs) | Every device speaks a fixed, limited language; extending what a machine can do means defining new instructions precisely | [Session 1](sessions/01_session-01/README.md) |
| 2 | April 11, 2026 | The Tower of Computer Languages — Binary, ASCII & Encoding (~4.5 hrs) | Computers speak only binary at the hardware level; numbers map to letters via ASCII; meaning is built layer by layer up a tower of languages | [Session 2](sessions/02_session-02/README.md) |
| 3 | April 19, 2026 | Path Descriptions, Binary Arithmetic & Assembly Language | The same path can have multiple valid solutions; binary carrying must be understood not memorized; Assembly adds a human-readable layer above binary using a small fixed vocabulary | [Session 3](sessions/03_session-03/README.md) |
| 4 | April 25, 2026 | The Full Language Tower — From Binary to Natural Language & First Hands-on with Code | The tower spans from binary to natural language; a single AI prompt can produce hundreds of lines of working code; every layer enforces its own rules and even one small deviation produces an error | [Session 4](sessions/04_session-04/README.md) |
| 4.5 | April 25 – May 02, 2026 | Home Activity — Build Something as a Family Using AI | The same prompt given to different AI tools produces different results; building a prompt as a group requires agreeing on what you want before you can ask for it clearly; self-driven building outside sessions is the goal, and it is already happening | [Home Activity 4.5](sessions/04.5_home_activity-04.5/README.md) |
| 5 & 6 | May 02 & May 09, 2026 | Revision through Structured Practice & Building with AI | Initial resistance to hard problems gives way to genuine effort with encouragement; binary addition needs consistent practice not just re-explanation; refining a prompt when it fails is a real skill; creative combinations emerge when students are nudged beyond the obvious | [Sessions 5 & 6](sessions/05_session-05-06/README.md) |
| 7 | May 16, 2026 | Revision Worksheet & First Python `print` (~1.5 hrs activity) | The robot can retrace existing lines — unstated assumptions block progress; assembly language needs a dedicated drill not just re-explanation; each `print` call produces one line of output; syntax errors on real code land the precision lesson better than any explanation | [Session 7](sessions/06_session-07/README.md) |
